5) HONEYCOMB SUPERSTRUCTURE / TOPOLOGY SEMANTICS (THE "ROLLED MAT" MECHANIC)
--------------------------------------------------------------------------------------------------------

TOPOLOGY STATE = function that mutates adjacency without changing SCW internal rules:
  topo_state = flat:
    - cells do NOT touch unless explicitly declared
    - portals disallowed by default; traversal paths may be empty/disconnected
  topo_state = rolled:
    - new membrane contacts appear: touches(SCW_i, SCW_j) edges added
    - portals may be minted *only* for touching pairs, gated by policy/budget

OBSERVABILITY RULE (high level):
  SCW_i can observe SCW_j iff:
    reachable(touches_graph, i -> j) AND permitted(GATE policy ∩ topo_epoch)

Maxey00 ROLE (bridge):
  - can open a portal edge ONLY when HSE says membranes touch AND Gate Issuer mints GATE# for that portal
  - otherwise must route via Traversal Router (multi-hop relay through intermediate cells)

6) RUNTIME ORDER (EXTENDED WITH MAP1/MAP2 + TOPOLOGY EPOCHS)
--------------------------------------------------------------------------------------------------------

(1) Claude Desktop starts -> user enables Maxey0 MCP server connection
(2) Claude MCP client discovers Maxey0: tools/resources/prompts
(3) User triggers Maxey0: scw.create(SCW0, determinism=strict|bounded, tick_logging=on)
(4) maxey18 HSE initializes topo_epoch=0, topo_state=flat (default)
(5) maxey186 Density Mapper emits $map1 baseline density snapshot -> stored in maxey44 artifact vault
(6) Maxey0 connects as MCP client to downstream planes; builds capability graph + policy envelopes
(7) Upload assistants registry (Maxeys.csv) -> creates SCW1..SCWn cells as needed; registers membranes
(8) For each assistant centroid:
      - Maxey00 superposes (observer lens)
      - maxey187 spawns drones inside target SCW to trace instruction-paths
      - maxey186 updates region densities from outside-of-cell view
(9) HSE triangulates: outside view (Maxey0) + inside lens (Maxey00) + drone paths -> emits $map2
(10) $map2 stored in maxey44 artifact vault; SCWs remain siloed by their capsule boundaries
(11) Optional: user toggles topo_state=rolled -> topo_epoch++ -> adjacency changes -> portals may become valid
(12) SCW0 seals run: CST + ledger entry + replay recipe + topo_epoch chain
